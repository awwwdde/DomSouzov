# ДОМ СОЮЗОВ — ДИЗАЙН-СПЕЦИФИКАЦИЯ САЙТА

Единый документ для всего проекта. Описывает визуальный язык, компонентную систему, структуру всех страниц, поведение версии для слабовидящих (по ГОСТ Р 52872-2019), правила футера по 152-ФЗ, моушн-язык, и изменения на бэкенде.

Стилистика — плотный издательский минимализм в духе журнала «Искусство Кино»: чёрно-белая база, тонкие линии-разделители, типографика капсом как главный носитель характера, точки-маркеры тегов, минимум декора. Один акцентный цвет — приглушённый изумрудный — используется крайне дозированно.

Этот документ — закон. Любая правка дизайна обновляет этот файл, а не наоборот.

## ЧАСТЬ 1. БАЗА

### 1.1 Принципы

**Композиция держится на сетке, а не на украшениях.** Все секции одной ширины (потолок 1600px, единые горизонтальные поля), один вертикальный ритм, левая направляющая — сквозная сверху донизу.

**Контент — типографика и фото.** Никаких карточек с тенями, градиентами, скруглениями, бликами. Разделители — тонкие линии 1px цвета `line`. Углы прямые (кроме кнопок-пилюль).

**Текст работает капсом.** Заголовки, метки, теги, числа дат — всё в `UPPERCASE` с трекингом. Только основной читаемый текст (абзацы, описания) — обычным регистром.

**Точки-маркеры — фирменный знак.** Любой тег категории начинается с круглой точки `accent` 6px. Этот элемент — визуальный «голос» сайта, появляется везде.

**Цвет молчит.** 95% сайта — чёрный текст на тёплом белом. Акцент-изумрудный показывается только в ключевых местах.

### 1.2 Сетка и пространство

**Контейнер:**
- Потолок ширины: `1600px`, центрируется на больших экранах.
- Горизонтальные поля от края экрана: моб `20px` / планшет `32px` / десктоп `48px`.
- Все секции используют компонент `<Section>` или `<Container>` из `components/Section.tsx`. Никаких `max-w-4xl/5xl/6xl` вразнобой.

**Вертикальный ритм** (пресет `spacing` у `<Section>`):
- `sm` → `py-14 md:py-20` — короткие служебные блоки
- `md` → `py-20 md:py-28` — **базовый** для большинства секций
- `lg` → `py-28 md:py-40` — манифест, крупные паузы
- `none` — полноширинные медиа-секции (история, hero)

Между заголовком и контентом секции — `mb-12 md:mb-16`.

**Разделители:** тонкая линия `1px solid var(--color-line)` (проп `bordered` у `<Section>`). Внутри сеток (афиша, новости) — те же линии между ячейками. Никаких рамок-боксов.

### 1.3 Цвет

| Токен | HEX | Использование |
|-------|-----|---------------|
| `paper` | `#f7f7f2` | основной фон сайта, тёплый белый |
| `paper-soft` | `#ffffff` | вторичный белый |
| `ink` | `#171717` | основной текст, тёмные блоки |
| `ink-soft` | `#4a4a45` | вторичный текст |
| `muted` | `#777770` | метки, кикеры, подписи |
| `line` | `#deded6` | все разделители |
| `accent` | `#2f5d50` | приглушённый изумрудный |

**Где появляется акцент:**
- короткая линия 40px перед кикером секции
- точка-маркер 6px у тегов категорий
- активная дата в календаре (кольцо, не заливка)
- ховер-цвет заголовков карточек
- активный пункт в списке (выбранный зал, активный фильтр)
- акцентная кнопка (Primary)

**Запрещено:** крупные изумрудные заливки, изумрудные градиенты, изумрудный фон секций.

**Тёмные блоки** (`bg-ink`) используются точечно как «смена сцены»: секция залов, полноширинная «история», футер. Нельзя ставить два тёмных блока подряд.

### 1.4 Типографика

**Шрифты** (остаются как сейчас): Roboto Condensed (`--font-heading`, 400/700, uppercase почти всегда), Manrope (`--font-sans`, 300–700, normal).

**Кегли и роли:**

| Роль | Шрифт | Кегль | Трекинг | Регистр |
|------|-------|-------|---------|---------|
| Mega H1 (логотип на главной) | heading 700 | `clamp(72px, 14vw, 220px)` | `0.01em` | upper |
| H1 страниц | heading 700 | `clamp(56px, 9vw, 140px)` | `0.02em` | upper |
| H2 секций | heading 700 | `clamp(40px, 6vw, 104px)` | `0.02em` | upper |
| H3 карточек | heading 700 | `clamp(14px, 1.1vw, 17px)` | `0.04em` | upper |
| Дата DD.MM на карточке | heading 700 | `clamp(22px, 2.2vw, 32px)` | `0.01em` | — |
| Кикер / Тег | sans 700 | `10px` | `0.22em` | upper |
| Основной абзац | sans 400 | `15–16px` | `0` | normal |
| Подпись / мета | sans 600 | `10–11px` | `0.18em` | upper |

Все заголовки `line-height: 0.86–0.98` — плотно, как блок.

**Кикер секции** (фирменный паттерн):
```
─── ●  СОБЫТИЯ
```
Короткая линия 40px `accent` + точка-маркер 6–8px `accent` + caps-метка `muted`. Появляется в каждой секции.

## ЧАСТЬ 2. КОМПОНЕНТЫ

### 2.1 Кнопки (три варианта, только пилюли)

**Primary (акцентная):** фон `accent`, текст `paper`, без рамки. Использование: главное CTA на экране («Смотреть афишу», «Больше о нас», «Купить билет»). **Не более одной** на видимом экране.

**Secondary (контурная тёмная):** фон `transparent`, рамка `1px solid ink`, текст `ink`. Использование: вторичное действие («Организаторам», «Вся афиша», «Все материалы»).

**Ghost (на тёмном):** фон `transparent`, рамка `1px solid rgba(247,247,242,0.45)`, текст `paper`. Использование: только на тёмных секциях.

Размеры: `padding: 12px 24px`, sans 700, `11px`, трекинг `0.16em`, uppercase, `min-height: 44px`.

### 2.2 Карточка афиши / новости («Искусство Кино»)

Основной компонент сайта. Используется в секциях афиши, новостей, на страницах списков.

**Структура:**
```
┌──────────────────────────┐
│      [ ИЗОБРАЖЕНИЕ ]     │   aspect-ratio: 4/5
├──────────────────────────┤   ← линия 1px line
│ ● СОБЫТИЯ        06.09   │   ← тег слева, дата справа
│                          │
│ ЗАГОЛОВОК КАРТОЧКИ       │   ← H3 capsed
│ В ДВЕ-ТРИ СТРОКИ         │
└──────────────────────────┘
```

**Правила:**
- Фото вверху, под ним — мета-строка (тег + дата) и заголовок. Текст **никогда не лежит на фото**.
- Тег слева: точка-маркер 6px `accent` + sans 700 caps `muted` 10px.
- Дата справа: heading 700 `clamp(22px, 2.2vw, 32px)`, формат `DD.MM`.
- Между фото и мета — `mt-4`, между мета и заголовком — `mt-3`.
- Заголовок — H3 caps, max 3 строки.
- Между карточками в сетке — **только линии** 1px `line` (вертикальные и горизонтальные).
- Ховер: заголовок → `accent`, фото `scale 1 → 1.04`, длительность 0.9с.
- Кликабельна целиком (`<Link>`).

**Сетка:** десктоп 3 кол, планшет 2 кол, моб 1 кол. Разделители-линии **между** ячейками.

### 2.3 Навигационный блок (Залы / Афиша / Новости)

Три больших текстовых блока-ссылки на главной.

```
┌──────────────────────────────┐
│                          ↗   │
│                              │
│  ЗАЛЫ                        │  H2 caps, нижне-левый
│                              │
└──────────────────────────────┘
```

- Высота ~280–360px, без фона
- Заголовок — heading 700 caps, `clamp(40px, 5vw, 84px)`, нижне-левый угол с паддингом
- Стрелка ↗ — SVG 32px, верхне-правый, паддинг 24–32px
- Разделители между блоками — только вертикальные линии
- Ховер: заголовок → `accent`, стрелка сдвигается на 4px по диагонали вправо-вверх

### 2.4 Бегущая строка

Существующий `Marquee.tsx`. Тёмная (`bg-ink`, текст `paper`), heading 700 `clamp(22px, 3.4vw, 46px)`, трекинг `-0.01em`, скорость 24–28с. Без реакции на скролл.

### 2.5 SectionHead (общий)

```
─── ● СОБЫТИЯ
БОЛЬШОЙ H2     [ кнопка действия → ]
```
Кикер + caps-метка слева сверху, ниже H2. Справа от H2 (на одной горизонтальной линии) — опциональная кнопка действия (Secondary). На мобайле H2 и кнопка в столбик.

### 2.6 Sticky-фильтр (для Афиши и Новостей)

Прилипающая шапка над списком: переключатель неделя/месяц + горизонтальная сетка дат + лента фильтров-тегов.

```
─────────────────────────────────────────────
  МАРТ 2026                  [Неделя | Месяц]
  
  ПН  ВТ  СР  ЧТ  ПТ  СБ  ВС
  04  05  06  07  08  09  10
  
  ● ВСЕ  ● КОНЦЕРТ  ● ФЕСТИВАЛЬ  ● ВЫСТАВКА
─────────────────────────────────────────────
```

- `position: sticky; top: 0` для всей шапки
- Активная дата — обведена кольцом `accent`
- Активный фильтр — заливка `ink`, текст `paper`, неактивный — `transparent` + `line` рамка
- Фильтры — пилюли с точкой-маркером `accent` слева

### 2.7 Lightbox (галерея и фотографии события)

При клике на фото:
- Открывается полноэкранный модал `bg-ink/95`, фото по центру `max-h: 90vh max-w: 90vw`
- Стрелки `←`/`→` по бокам для навигации
- Крестик в правом верхнем `×`
- Подпись фото (если есть) — снизу sans 400 14px `paper/80`
- Закрытие — клик вне фото, Esc, или крестик
- Для видео — встроенный плеер (controls), но без autoplay

## ЧАСТЬ 3. СТРАНИЦЫ

### 3.1 ГЛАВНАЯ (`/`)

```
1. HERO                    H1 + кикер столбиком + видео + кнопка над видео
2. ABOUT-INTRO             короткий текст + 2 кнопки
3. MARQUEE                 бегущая строка
4. UPCOMING EVENTS         сетка карточек 3×N (макс 6, закреплённые сначала)
5. CTA-BANNER              призыв выбрать мероприятие
6. NEWS                    сетка карточек 3×N (макс 6, закреплённые сначала)
7. NAV-TRIO                три блока: Залы / Афиша / Новости с ↗
8. PARTNERS                логотипы из CMS
9. FOOTER                  полный по 152-ФЗ
```

**HERO** (не полноэкранный, как обложка журнала):
```
┌─────────────────────────────────────────────────────────┐
│  ┌──────────────────────────┐    ┌─────────────────┐   │
│  │                          │    │ КУЛЬТУРА        │   │
│  │       ДОМ СОЮЗОВ         │    │ АРХИТЕКТУРА     │   │
│  │  (mega H1, ~80% ширины)  │    │ СОБЫТИЯ         │   │
│  └──────────────────────────┘    └─────────────────┘   │
│                                                         │
│  ┌──────────────────────────────────────┐               │
│  │                                       │ ┌──────────┐│
│  │                                       │ │ СМОТРЕТЬ ││
│  │           [ ВИДЕО 16:9 ]              │ │ АФИШУ →  ││
│  │                                       │ └──────────┘│
│  │                                       │  top 30px   │
│  │                                       │  right 30px │
│  └──────────────────────────────────────┘               │
└─────────────────────────────────────────────────────────┘
```
- H1 «ДОМ СОЮЗОВ» Mega-H1, на ~80% ширины контейнера слева.
- Кикер 3 строки столбиком справа, выровнен по правому краю, центрирован по вертикали относительно H1 (через flex `items-center`).
- Видео ниже H1, ширина = ширине контейнера, `aspect-ratio: 16/9`, `object-fit: cover`, `autoplay/loop/muted/playsInline`, плакат-постер обязателен.
- Кнопка «Смотреть афишу» — `position: absolute; top: 30px; right: 30px` относительно блока видео. Primary.
- Отступ между H1 и видео — `mt-10 md:mt-16`.
- **Без параллакса, без зума, без вуалей.** Видео играет как просто видео.

**ABOUT-INTRO:** короткий блок под видео. Слева тезис capsed (`clamp(28px, 3vw, 48px)`), справа абзац `ink-soft`. Под обеими колонками — две кнопки: Primary «Больше о нас», Secondary «Организаторам». Без фото.

**MARQUEE:** существующий компонент, темы вроде «КОЛОННЫЙ ЗАЛ · С 1784 ГОДА · БОЛЬШАЯ ДМИТРОВКА 1 · КУЛЬТУРНАЯ ПЛОЩАДКА МОСКВЫ».

**UPCOMING EVENTS / NEWS:** кикер «События» / «Новости» + H2 + кнопка-действие. Сетка карточек 3 кол. **Логика выбора 6 карточек:**
- Если есть события/новости с флагом `is_pinned` (новый булев флаг, см. Часть 5) — они идут первыми по `order` пина.
- Остальные слоты заполняются последними добавленными (по `created_at` DESC).
- Итого до 6 штук.

**CTA-BANNER** (между EVENTS и NEWS):
- Полноширинная секция ~50vh
- Центрированный H2 capsed: «ВЫБЕРИТЕ СВОЁ МЕРОПРИЯТИЕ»
- Под ним Primary кнопка «Смотреть афишу»
- Фон — приглушённое фото из CMS или сплошной `ink`

**NAV-TRIO:** три блока Залы / Афиша / Новости в ряд (см. 2.3). На мобайле — столбиком.

**PARTNERS:** существующий `PartnersSection.tsx`. Кикер «Партнёры» + сетка логотипов. Логотипы по умолчанию `filter: grayscale + opacity: 0.55`, на ховере проявляются.

### 3.2 АФИША (`/events`)

**Структура:**
```
─── ● АФИША
БОЛЬШОЙ H1 «АФИША»

[ STICKY-БЛОК: календарь + фильтры ]   ← position: sticky, top: header-height
[ список событий]
```

**Sticky-блок** (компонент 2.6):
- Шапка с месяцем + переключатель неделя/месяц
- Сетка дат (7 в неделе, ~30 в месяце), активная дата с кольцом `accent`
- Лента фильтров-тегов под датами

**Список событий** — карточки в стиле «Искусство Кино», но **горизонтальные полосы** (другой вариант 2.2 для крупных списков):

```
┌────────────────┬───────────────────────────────────────┐
│                │  ● КОНЦЕРТ                    14.03   │
│  [ФОТО]        │                                       │
│                │  НАЗВАНИЕ МЕРОПРИЯТИЯ                 │
│                │  Колонный зал · 19:30                 │
│                │                                       │
│                │  [Купить билет] (если has_ticket)     │
└────────────────┴───────────────────────────────────────┘
─────────────────────────────────────────────────────────
┌────────────────┬───────────────────────────────────────┐
│                │  ● ФЕСТИВАЛЬ                  18.03   │
│  [ФОТО]        │  ...                                  │
│                │                                       │
└────────────────┴───────────────────────────────────────┘
```

- Каждая полоса прилипает при скролле (паттерн из текущего `FullCalendar`) — карточки наслаиваются друг на друга.
- Слева фото `aspect: 4/3`, справа: тег + дата (как в карточке), название, зал/время, кнопка билета.
- Между полосами — линия `line`.
- При выборе даты в sticky-календаре — список фильтруется по этой дате.
- При выборе тега — список фильтруется по тегу.

### 3.3 НОВОСТИ (`/news`)

**Зеркало страницы Афиши:** тот же sticky-календарь с фильтрами по тегам, такой же список карточек-полос. Дата у новости — `created_at`, при выборе даты в календаре показываются новости этого дня.

### 3.4 СОБЫТИЕ (`/events/:id`) и НОВОСТЬ (`/news/:id`)

Единая раскладка для обеих сущностей.

```
┌─────────────────────────────────────────────────┐
│  ─── ● СОБЫТИЯ                       14.03.2026 │
│  БОЛЬШОЙ ЗАГОЛОВОК СОБЫТИЯ                      │
│  короткое описание-лид (1-2 строки, опционально)│
├─────────────────────────────────────────────────┤
│                                                 │
│  ┌───────────────────┐  ИНФО МЕРОПРИЯТИЯ        │
│  │                   │  Зал: Колонный           │
│  │   [ ОБЛОЖКА ]     │  Время: 19:30            │
│  │   aspect 4/3      │  Возраст: 6+             │
│  │                   │  Цена: 1500–5000 ₽       │
│  │                   │                          │
│  └───────────────────┘  [Купить билет] (опц.)   │
│                                                 │
├─────────────────────────────────────────────────┤
│                                                 │
│  ┌────────────────┐    ТЕКСТ ОПИСАНИЯ          │
│  │ [галерея 1]    │    Длинный текст           │
│  │ aspect 4/5     │    мероприятия...           │
│  └────────────────┘                             │
│                          ← это всё sticky,      │
│  ┌────────────────┐         не уезжает          │
│  │ [галерея 2]    │                             │
│  └────────────────┘                             │
│                                                 │
│  ┌────────────────┐                             │
│  │ [галерея 3]    │                             │
│  └────────────────┘                             │
│  ...                                            │
└─────────────────────────────────────────────────┘
```

**Раскладка:**
- **Шапка:** кикер «События»/«Новости» + дата справа, под ним H1, под ним опциональный лид. Линия снизу.
- **Верхний блок (обложка + инфо):** обложка слева, инфо справа на одной горизонтали.
- **Нижний блок (галерея + текст):** левый столбец — фотки галереи (поочерёдно вниз), правый столбец — sticky `top: header + 24px`, содержит инфо-блок (повтор из верхнего) + текст описания. Sticky продолжается до конца секции.
- **Анимация галереи:** каждая фотка появляется через `whileInView` (mask-reveal + opacity), задержка 0.15с между фото, easing `EASE_DS`. Через Lenis скролл плавный.

**Кнопка билета:**
- Если `event.has_ticket === true` И `event.ticket_url` задан → Primary кнопка «Купить билет» с переходом по `ticket_url` в новой вкладке.
- Если `has_ticket === false` → блок «Вход свободный» / «По регистрации» (текст из CMS) или просто отсутствие кнопки.
- На странице новости кнопки билета нет вообще.

**Клик по фотографии галереи** → открывает Lightbox (см. 2.7) с навигацией по всем фото галереи.

### 3.5 ЗАЛЫ (`/halls`)

## ЧАСТЬ 6. ВЕРСИЯ ДЛЯ СЛАБОВИДЯЩИХ (ГОСТ Р 52872-2019)

### 6.1 Что это и зачем

ГОСТ Р 52872-2019 — это российский стандарт доступности интернет-ресурсов, основанный на международном WCAG 2.1. Для государственных учреждений соответствие обязательно (закон 419-ФЗ). Дом Союзов под это попадает.

Текущая реализация (атрибут `data-vision="true"` на `<html>` + гашение анимаций + смена темы и шрифта) — это не полноценная версия для слабовидящих, а косметическая адаптация. Нужно переработать в **полноценный отдельный визуальный режим**, где сайт превращается в чистый, высококонтрастный, одноколоночный документ.

### 6.2 Панель управления

В Header — кнопка-иконка `Eye` (`lucide-react`). Клик → открывает плавающую панель управления Vision Mode (модалка или dropdown):

```
┌──────────────────────────────────────────────┐
│  ВЕРСИЯ ДЛЯ СЛАБОВИДЯЩИХ                  ×  │
├──────────────────────────────────────────────┤
│                                              │
│  ☐ ВКЛЮЧИТЬ ВЕРСИЮ                           │
│                                              │
│  РАЗМЕР ШРИФТА                               │
│  [ Обычный | Крупный | Очень крупный ]       │
│                                              │
│  ЦВЕТОВАЯ СХЕМА                              │
│  [ Чёрный на белом | Белый на чёрном |       │
│    Чёрный на бежевом | Синий на белом ]      │
│                                              │
│  МЕЖСТРОЧНЫЙ ИНТЕРВАЛ                        │
│  [ Стандартный | Увеличенный ]               │
│                                              │
│  ☐ ОТКЛЮЧИТЬ ИЗОБРАЖЕНИЯ                     │
│  ☐ ОТКЛЮЧИТЬ АНИМАЦИИ                        │
│                                              │
│  [Сбросить настройки]                        │
└──────────────────────────────────────────────┘
```

Настройки сохраняются в `localStorage` (`domsouzov-vision-settings`, JSON):
```json
{
  "enabled": true,
  "fontSize": "large",
  "colorScheme": "black-on-white",
  "lineSpacing": "increased",
  "noImages": false,
  "noAnimations": true
}
```

Соответственно на `<html>` ставятся data-атрибуты:
- `data-vision="true"`
- `data-vision-font="normal|large|xlarge"`
- `data-vision-theme="bw|wb|bb|by"` (black-white, white-black, black-beige, blue-white)
- `data-vision-line="normal|increased"`
- `data-vision-no-images="true"` (опционально)
- `data-vision-no-animations="true"` (опционально, по умолчанию `true` при включении режима)

### 6.3 Что делает Vision Mode (требования ГОСТ Р 52872-2019)

Когда `data-vision="true"` активно — применяются **все** правила ниже одновременно:

**Типографика:**
- Минимум 16px базовый шрифт (`normal`)
- `large`: 20px, `xlarge`: 24px
- Межстрочный интервал: `1.5` (`normal`) или `1.8` (`increased`)
- Только sans-serif (Manrope), без caps в тексте — только в коротких метках
- Заголовки крупнее стандарта на 1 шаг
- Минимальный кегель кнопок 16px, мет — 14px

**Цвет и контраст:**
- Только 4 высококонтрастные пары (см. `colorScheme`)
- Никаких полутонов: убираются `ink-soft`, `muted`, `line` (заменяются на цвет контура темы)
- Никаких градиентов, теней, прозрачностей
- Контраст не ниже 7:1 (уровень AAA WCAG)

**Композиция и интерактив:**
- **Одноколоночная вёрстка** — все секции в столбик, никаких гридов из 2-3 колонок (контент линейный сверху вниз)
- Минимальный размер интерактивных зон 48×48px (ГОСТ Р 52872-2019 + Material Design)
- Все ссылки подчёркнуты `text-decoration: underline`
- Фокус обведён жирной рамкой `3px solid` цвета контура темы, `outline-offset: 4px`
- Все элементы должны быть досягаемы клавиатурой (`tab`), порядок фокуса логичен

**Контент:**
- **Видео не воспроизводится автоматически** (если есть — заменяется на постер + ссылка «Смотреть видео»)
- **Декоративные изображения скрыты** (точки-маркеры, кикеры-линии, бегущая строка, параллакс)
- **Контентные изображения остаются**, но с обязательным `alt`, под фото — подпись sans 400 14px
- Если `noImages: true` — все изображения скрываются, остаётся `alt`-текст
- Бегущая строка `Marquee` отключается полностью (либо превращается в статичный список)

**Анимации:**
- Все анимации Framer Motion (через `useReducedMotionActive()`) — отключены
- CSS `transition` и `animation` — `none !important` (уже есть в `index.css`)
- Параллакс, mask-reveal, scroll-driven — отключены
- Lenis (плавный скролл) — отключён, нативный скролл
- Прелоадер — пропускается (см. 6.5)

**Структурно:**
- Главная упрощается: убираются NAV-TRIO (заменяются обычным списком ссылок), CTA-BANNER упрощается (только заголовок и ссылка), Marquee скрыт
- Hero на главной: H1 + текстовое описание + ссылка «Смотреть афишу» (без видео, без кикера-столбика)
- На детальных страницах sticky-поведение отключено (правый столбец просто прокручивается с остальной страницей)
- Cookie-баннер всегда виден до клика, без анимации

### 6.4 Цветовые схемы Vision Mode

| Схема | Фон | Текст | Контур/линии | Ссылки |
|-------|-----|-------|--------------|--------|
| `bw` (по умолчанию) | `#ffffff` | `#000000` | `#000000` | `#0033cc` |
| `wb` (инверсия) | `#000000` | `#ffffff` | `#ffffff` | `#9cd8ff` |
| `bb` (чёрный на бежевом) | `#f4ecd8` | `#1a1a1a` | `#1a1a1a` | `#003366` |
| `by` (синий на белом) | `#ffffff` | `#003366` | `#003366` | `#003366` |

```css

  --color-paper: #ffffff;
  --color-paper-soft: #ffffff;
  --color-ink: #000000;
  --color-ink-soft: #000000;
  --color-muted: #000000;
  --color-line: #000000;
  --color-accent: #0033cc;
}
/* и так для каждой темы */
```

### 6.5 Прелоадер и Vision Mode

- Если Vision Mode **включён до загрузки** (флаг в `localStorage`) — прелоадер **не показывается** вообще (мгновенно `onComplete`).
- Если Vision Mode выключен — прелоадер показывается **при каждой загрузке страницы** (вариант A: при `F5` или прямом заходе по URL, но не при внутренней навигации). Это убирает текущий `sessionStorage` ключ `ds_preloader_shown`.

### 6.6 Доступ к Vision Mode без визуального интерфейса

Для пользователей, которые уже используют скринридеры или экранные лупы — должен быть прямой URL: `?vision=on`, который включает режим автоматически при первой загрузке. Это требование ГОСТ.

## ЧАСТЬ 7. МОУШН-ЯЗЫК

Сдержанный. Только эти приёмы:

1. **Появление при скролле:** `opacity 0→1, y 16→0`, длительность 0.7–0.9с, easing `cubic-bezier(0.22, 1, 0.36, 1)`, `viewport once: true`.
2. **Mask-reveal для крупных заголовков:** clip-path шторка снизу вверх, длительность 1.0с.
3. **Ховер карточек:** заголовок → `accent`, фото `scale 1 → 1.04`, длительность 0.9с.
4. **Ховер кнопок:** лёгкий подъём `y: -2px`, 0.2с.
5. **Ховер NAV-блоков:** стрелка ↗ сдвиг 4px вправо-вверх, заголовок → `accent`.
6. **Sticky-карточки в Афише/Новостях:** Lenis обеспечивает плавный скролл, карточки наслаиваются через `position: sticky`.
7. **Галерея на детальной странице:** каждое фото — `whileInView`, mask-reveal + opacity, задержка 0.15с между.
8. **Переходы страниц:** `opacity` + `y` через `AnimatePresence`, 0.5с.

**Запрещено:** spring на крупных элементах, бесконечные анимации кроме Marquee, параллакс на каждом блоке, scale > 1.05 на ховере, velocity-driven эффекты.

**Глобальное отключение** через `useReducedMotionActive()` при:
- `prefers-reduced-motion: reduce` (системная настройка пользователя)
- `data-vision="true"` (Vision Mode)
- `data-vision-no-animations="true"` (подопция Vision Mode)
