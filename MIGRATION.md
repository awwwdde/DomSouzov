# Перенос Дома Союзов на новый домен/хост (с данными БД)

Главное, что нужно понимать: **все данные, включая загруженные картинки, PDF и видео,
лежат в базе данных** (таблица `media_files`, бинарь в `bytea`). На диске файлов нет.
Поэтому «перенести БД» = перенести вообще всё. Ничего отдельно копировать из `uploads/`
не нужно.

Смена домена не требует правок кода: canonical/og/sitemap строятся от переменной
`SITE_URL`, а `index.html` переписывается сервером на лету. Достаточно поменять env.

---

## 1. Подготовка нового окружения (env)

На новом хосте задайте переменные (в panel — через UI проекта, на VPS — в `backend/.env`):

```
SITE_URL=https://НОВЫЙ-ДОМЕН
CORS_ORIGINS=https://НОВЫЙ-ДОМЕН
SECRET_KEY=<32+ случайных символа>           # НЕ дефолтный
BOOTSTRAP_ADMIN_EMAIL=<ваш логин>            # или ADMIN_EMAIL для локалки
BOOTSTRAP_ADMIN_PASSWORD=<надёжный пароль>   # или ADMIN_PASSWORD
SEED_DEMO_CONTENT=false                       # чтобы не подсеялся демо-контент
DATABASE_URL=<см. ниже под выбранный путь>
```

> На awwwdde-panel `DATABASE_URL` и `SECRET_KEY` панель проставляет сама — тогда
> руками задаются только `SITE_URL`, `CORS_ORIGINS`, `SEED_DEMO_CONTENT` и админ.

## 2. Выбор пути переноса

### Путь A — новый хост на PostgreSQL, и БД доступна с вашей машины
Самый простой. С локальной машины (где наполнена `domsoyuzov.db`):

```bash
cd backend
# venv активирован
set SOURCE_DATABASE_URL=sqlite:///./domsoyuzov.db
set TARGET_DATABASE_URL=postgresql+psycopg2://USER:PASS@НОВЫЙ_ХОСТ:5432/domsoyuzov
python migrate_to_postgres.py
```

Скрипт создаёт схему и переносит **все** таблицы, включая `media_files`
(картинки/PDF/видео) и `newsletter_subscribers`, затем выравнивает sequence.
Цель должна быть **пустой** (скрипт не идемпотентен — на непустой БД будут конфликты id).

### Путь B — новый хост на PostgreSQL, но БД НЕ доступна снаружи (фаервол)
Перенос делаем на сервере:

1. Скопируйте на сервер файл `backend/domsoyuzov.db` (например, `scp`).
2. На сервере, рядом с кодом бэкенда:
   ```bash
   set SOURCE_DATABASE_URL=sqlite:////полный/путь/domsoyuzov.db
   set TARGET_DATABASE_URL=postgresql+psycopg2://USER:PASS@localhost:5432/domsoyuzov
   python migrate_to_postgres.py
   ```
3. Удалите скопированный `domsoyuzov.db` с сервера после успешного переноса.

### Путь C — новый хост на SQLite с постоянным диском (volume)
Если новый хост монтирует постоянный том (НЕ awwwdde-panel — там диск не
сохраняется между деплоями):

1. Скопируйте `backend/domsoyuzov.db` в каталог данных на новом хосте.
2. `DATABASE_URL=sqlite:////путь/к/domsoyuzov.db`
3. Готово — файл несёт в себе всё, включая медиа.

> ⚠️ Если у нового хоста нет постоянного диска — SQLite потеряется при первом
> же редеплое. Тогда только PostgreSQL (Путь A/B).

## 3. Первый старт на новом хосте

При старте контейнер выполняет `python seed.py` (создаёт недостающие таблицы и
докидывает дефолтных админа/настройки), затем поднимает uvicorn. Поскольку данные
уже перенесены, демо-контент не добавится (БД не пустая), а `SEED_DEMO_CONTENT=false`
страхует дополнительно.

---

## 4. Проверка после переноса (smoke-test)

- [ ] `GET /healthz` → `{"status":"ok"}`
- [ ] Открыть сайт — события/новости/залы/галерея на месте.
- [ ] Открыть любую загруженную картинку: `GET /uploads/<имя>` отдаётся (значит медиа доехало).
- [ ] Зайти в `/admin` под прод-логином, проверить что контент виден и редактируется.
- [ ] `GET /sitemap.xml` и `GET /robots.txt` содержат **новый** домен.
- [ ] Открыть страницу события — в исходном HTML правильные `<title>`/`og:url`
      с новым доменом (это видит краулер).

---

## 5. После запуска (SEO)

- Submit `https://НОВЫЙ-ДОМЕН/sitemap.xml` в Яндекс.Вебмастер и Google Search Console.
- Если старый домен закрывается — настроить 301-редирект со старого на новый
  (на уровне старого хоста/DNS), чтобы не потерять индексацию.
